# 洗練された UI デザインガイド

PolygonScan の DIM モードおよび Apple Human Interface Guidelines (HIG) を参照した、プロフェッショナルで美しい Web アプリケーションの UI 実装ガイド。

## 参照ソースの特徴

### PolygonScan / Etherscan

- **Light / Dim / Dark** の3モードを提供
- **Dim モード**: Light と Dark の中間。コントラストを抑え、長時間閲覧に適した柔らかい表示
- トグル: ロゴ横の Day/Night スイッチで切替
- 情報階層が明確で、ブロックチェーンなどの複雑なデータを簡潔に表示
- 読みやすさとプロフェッショナルさを重視

### Apple Human Interface Guidelines

- **Clarity（分かりやすさ）**: 一目で理解できる UI
- **Deference（コンテンツ優先）**: 装飾より内容を前面に
- **Depth（階層感）**: 視覚的レイヤーで情報構造を伝える
- **Consistency（一貫性）**: プラットフォーム内外で統一された体験

## 1. カラーパレット

### 避けるべきこと

- 鮮やかすぎる青（`#2563EB` 等）をアクセントとして多用
- 鮮やかな緑のタグ・ pill
- フラットで彩度の高い配色
- 背景と要素のコントラスト差が極端なダークモード

### 推奨カラー（DIM / 洗練されたダーク）

| 用途           | 推奨値（例）        | 備考                                      |
|----------------|---------------------|-------------------------------------------|
| 背景（メイン） | `#12161C` 〜 `#1C2128` | PolygonScan 風の落ち着いた暗色             |
| 背景（カード） | `#1E242B` 〜 `#252D37` | メインよりわずかに明るい                   |
| テキスト（主） | `#E4E6EA` 〜 `#F0F1F3` | 白すぎない、疲れにくい白                   |
| テキスト（補） | `#9CA3AF` 〜 `#AEB3BB` | グレー寄りの控えめな補助テキスト           |
| テキスト（弱） | `#6B7280`            | tertiary 相当                             |
| アクセント     | `#60A5FA` 〜 `#93C5FD` | ミュートな青（Tailwind `blue-400` 相当）  |
| リンク・強調   | `#67C6C0` 〜 `#7DD3D0` | 控えめなシアン・ティール                  |
| 成功           | `#6EE7B7`            | ミュートな緑（`emerald-400` 相当）        |
| 警告           | `#FCD34D` 〜 `#FDE68A` | くすんだアンバー                          |
| エラー         | `#F87171`            | ミュートな赤（`red-400` 相当）           |
| ボーダー       | `#2D3748` 〜 `#374151` | かすかな区切り線                          |

### Tailwind での実装例

```css
/* カスタム DIM  palette */
--bg-main: #12161C;
--bg-card: #1E242B;
--text-primary: #E4E6EA;
--text-secondary: #9CA3AF;
--accent-muted: #60A5FA;
--border-subtle: #2D3748;
```

## 2. タイポグラフィ

- **フォント**: SF Pro に近いニュートラルな Sans-serif
  - Web: `"SF Pro Text", "SF Pro Display", -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`
  - 日本語: `"Hiragino Sans", "Hiragino Kaku Gothic ProN", "Noto Sans JP", sans-serif`
- **サイズ階層**: 明確なステップ（例: 12px / 14px / 16px / 20px / 24px）
- **行の高さ**: 1.4 〜 1.6 で可読性確保
- **ウェイト**: `font-normal` を基本、見出し・ラベルは `font-medium` / `font-semibold`

---

## 3. 余白・ spacing

- **Deference の原則**: 要素を詰め込みすぎず、余白で呼吸感を出す
- **推奨**: `p-4` (16px) をカードの基本、重要な区切りは `p-6` / `gap-6`
- テーブル: セルパディング `py-3 px-4`、行間は十分に
- セクション間: `space-y-6` または `gap-6` でまとまりを明確に

---

## 4.  depth（階層感）

- **影**: 控えめに。`shadow-sm` / `shadow` 程度。極端なドロップシャドウは避ける
- **ボーダー**: 細く薄い。`border border-slate-700/50` など半透明を活用
- **背景の段階**: メイン → カード → 入力欄の順にわずかに明るくし、重なりを表現
- **カード**: `rounded-lg` 〜 `rounded-xl`、角を丸めて柔らかさを出す

---

## 5. コンポーネント別ガイド

### ボタン

- プライマリ: ミュートなアクセント色、`rounded-lg`、`px-4 py-2.5`
- セカンダリ: 背景を薄く `bg-slate-700/50`、ホバーで `bg-slate-600/50`
-  ghost: 枠なし、ホバー時のみ背景表示

### タグ・ pill

- 背景: `bg-slate-600/60` や `bg-slate-500/40` など半透明のグレー
- テキスト: `text-slate-200` 〜 `text-slate-300`
- 鮮やかな青・緑の単色背景は避け、必要なら `blue-500/30` のように彩度を落とす

### テーブル

- ヘッダー: `text-slate-400` + `font-medium` + `border-b border-slate-700/60`
- 行: ホバーは `hover:bg-slate-800/40` 程度の控えめな変化
- ゼブラストライプは避けるか、ごく薄く（`bg-slate-800/20` など）

### 入力欄

- 背景: カードよりわずかに暗く `bg-slate-800/50`
- ボーダー: `border-slate-600`、フォーカスは `ring-2 ring-blue-500/50`
- placeholder: `text-slate-500`

### JSON / コード表示

- キー: オレンジよりミュートに（例: `#D4A574`）
- 文字列: 薄い緑 `#9FDFBF` または白に近いグレー
- 数値: 薄い青
- 警告バー: 背景 `#322A1E`、テキスト `#FDD68A` など、目立ちすぎない警告色

## 6. 実装チェックリスト

UI 改善時は以下を確認する:

1. **色**: 鮮やかな色の多用を避け、ミュートで統一されたパレットか
2. **余白**: 詰め込みすぎず、Deference を満たしているか
3. **階層**: 重要度に応じた視覚的区別（背景・テキスト・アクセント）があるか
4. **タイポ**: フォント・サイズ・ウェイトに一貫性があるか
5. **コントラスト**: テキストと背景の比が 4.5:1 以上か（アクセシビリティ）
6. **タッチ領域**: ボタン・リンクは最低 44×44px か（モバイル対応時）

## 7. 既存 ui-design.mdc との関係

- 本スキルは **推奨スタイル** を拡張・補足する
- `ui-design.mdc` の Slate / Indigo をそのまま使う場合も、**彩度を落とし**、Dim 風に調整する
- 例: `indigo-600` → `indigo-500/80` または `blue-500/70`
